# zone-api вЂ” Implementation Plan

## Context

`zone-api` is the foundational zone intelligence service. It detects supply/demand zones from candlestick data, monitors live price against those zones, and publishes typed events over WebSocket. It has no broker awareness and no trade logic.

**Data dependency:** zone-api consumes candlestick data and real-time price ticks from `symbol-universe` over HTTP/WS. It does **not** hold its own DXLink client or TastyTrade API connection.

Three transports from a single process:
- **HTTP REST** вЂ” query zones, scan symbols, serve live chart pages
- **WebSocket** вЂ” real-time zone lifecycle + price action event stream
- **MCP (stdio)** вЂ” direct Claude agent access via tool calls

**Repo:** `/Users/chrismikelson/Github/zone-api/`
**Port:** 3001
**Depends on:** `symbol-universe` (port 3005) вЂ” must be running

## SQLite Schema

```sql
CREATE TABLE zones (
  id INTEGER PRIMARY KEY, symbol TEXT NOT NULL, timeframe TEXT NOT NULL,
  direction INTEGER NOT NULL, type INTEGER NOT NULL,
  proximal REAL NOT NULL, distal REAL NOT NULL, confidence REAL NOT NULL,
  entry_price REAL, stop_price REAL, target_price REAL,
  start_ts INTEGER NOT NULL, end_ts INTEGER NOT NULL,
  detected_at INTEGER NOT NULL, is_fresh INTEGER DEFAULT 1
);
CREATE INDEX idx_zones_symbol ON zones(symbol, timeframe, is_fresh);

CREATE TABLE confluent_zones (
  id INTEGER PRIMARY KEY, symbol TEXT NOT NULL, timeframes TEXT NOT NULL,
  direction INTEGER NOT NULL, proximal REAL NOT NULL, distal REAL NOT NULL,
  combined_confidence REAL NOT NULL, entry_price REAL, stop_price REAL,
  target_price REAL, price_inside INTEGER DEFAULT 0, computed_at INTEGER NOT NULL
);
CREATE INDEX idx_confluent_symbol ON confluent_zones(symbol, combined_confidence DESC);
CREATE INDEX idx_confluent_active ON confluent_zones(price_inside, combined_confidence DESC);

CREATE TABLE zone_touches (
  id INTEGER PRIMARY KEY, zone_id INTEGER REFERENCES confluent_zones(id),
  symbol TEXT NOT NULL, price REAL NOT NULL,
  event TEXT NOT NULL, -- zone_created|zone_updated|zone_expired|zone_entered|zone_exited|zone_breached
  outcome TEXT,        -- hold|breach (filled in retroactively by ML pipeline)
  touched_at INTEGER NOT NULL
);
-- No trades table вЂ” owned by execution-engine
```

The `outcome` column on `zone_touches` is the ML training label for `ml-services`.

## REST API

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/symbols/:symbol` | Start monitoring: fetch candles from symbol-universe, detect zones, open price stream |
| `DELETE` | `/symbols/:symbol` | Stop monitoring |
| `GET` | `/symbols` | All monitored symbols with current price + zone count |
| `GET` | `/scan/:symbol` | Proxy to `symbol-universe GET /metrics/:symbol` вЂ” IV rank, IVx, earnings |
| `GET` | `/zones/:symbol` | Fresh zones, `?timeframe=1d` filter optional |
| `GET` | `/zones/:symbol/confluent` | Multi-TF confluence zones sorted by confidence |
| `GET` | `/zones/top` | Top N zones across all symbols `?limit=10&direction=supply` |
| `GET` | `/zones/active` | All zones where `price_inside=1` right now |
| `GET` | `/chart/:symbol` | Live ECharts candlestick + zone overlay HTML page |

## Core Services

### `universeClient.ts`
HTTP + WebSocket client pointed at `symbol-universe` (configurable via `SYMBOL_UNIVERSE_URL`).

```typescript
// HTTP
fetchCandles(symbol, timeframe): Promise<Candle[]>
  в†’ GET http://symbol-universe:3005/candles/:symbol/:timeframe

fetchMetrics(symbol): Promise<MarketMetrics>
  в†’ GET http://symbol-universe:3005/metrics/:symbol

// WebSocket
subscribePrice(symbol, onTick: (price, bid, ask, ts) => void): () => void
  в†’ ws://symbol-universe:3005/stream/:symbol вЂ” listens for { type: 'price' } events
```

### `zoneService.ts`
1. Call `universeClient.fetchCandles(symbol, '1d' | '60m' | '15m')` for each timeframe
2. Run `identifyZones()` + `filterFreshZones()` from `@cmike444/supply-and-demand-zones` on each TF
3. Confluence: zones from different TFs with overlapping `[distal, proximal]` ranges в†’ merged `ConfluentZone`
4. `combined_confidence` = avg + 0.1 per extra TF
5. Emit `zone_created` / `zone_updated` / `zone_expired`
6. Refresh every 60 min

### `priceService.ts`
Calls `universeClient.subscribePrice(symbol, handler)`. On each tick:
- Price enters `[distal, proximal]` в†’ `zone_entered`, `price_inside=1`
- Price exits without breach в†’ `zone_exited`, `price_inside=0`
- Price closes through `distal` в†’ `zone_breached`, `is_fresh=0`

No direct DXLink dependency вЂ” price stream sourced from symbol-universe.

## `/chart/:symbol` вЂ” Live Chart

Extends `SupplyAndDemand/scripts/visualizeZones.ts` ECharts template:
1. Load: fetch `/zones/:symbol` + `/scan/:symbol` from own REST
2. Connect to WS `/stream/:symbol`
3. `candle` в†’ append/update chart data
4. `price` в†’ update price marker line
5. `zone_entered` в†’ highlight zone overlay
6. `zone_created` / `zone_updated` в†’ re-render overlays
7. `zone_breached` в†’ remove overlay
